# __NVIDIA_HDV__
__用于配合浏览器DevTools工具实现获取NVIDIA历史驱动的自动化程序__

> [!WARNING]
> __此项目系 [`NVIDIA_HDA`](https://github.com/FeiLingshu/NVIDIA_HDA) 的重置版本__
> - __自 `2026.4.9` 起，NVIDIA官网全面启用新的前端页面，旧项目已无法正常工作__
> - __由于 `Manifest-V3` 及 `CSP` 的安全限制，无法动态修改网络站点的JS代码__
> - __新版NVIDIA官网的后端代码中使用了无法在 `content` 世界中运行的代码，旧项目中使用的运行策略经测试已无法正常工作__
> 
> __基于上述原因，本项目将不再集成于浏览器扩展中，而是利用浏览器 `DevTools` 工具（即F12开发者工具），直接在运行时动态替换JS文件，以实现相关功能__

---

### __改进说明__
__较旧项目，本重置版本对功能进行了以下修改动__
- [x] __保留了强制检索 `Standard` 版本的功能__
   - __不建议使用此功能（可能会导致检索不到任何驱动），如有需要请将系统版本设置为 `win7` 再进行搜索__
- [ ] __移除了处于测试状态的 `Beta` 版本驱动的检索功能（目前依旧没搞明白实现方式）__
- [x] __添加了可配置的驱动类别筛选项，现允许仅搜索 `Game Ready` 驱动或 `Studio` 驱动__
- [x] __保留了可配置的 `version` 和 `release` 两个额外参数（前者用于搜索特定版本，后者用于搜索特定release系列）__

### __使用前检查__
__程序需要 [`.Net Framework 4.8`](https://dotnet.microsoft.com/zh-cn/download/dotnet-framework/net48) 运行环境__
> - __如需自行编译，请下载「 生成应用 - 开发包 」列下的内容__
> - __如仅使用，请下载「 生成应用 - 运行时 」列下的内容__
> - __下载「 运行时 」时，推荐优先使用「 Web 安装程序 」进行安装__

___⚠️ 使用「 脱机安装程序 」进行安装时，推荐下载所在地区的「 语言包 」，以确保程序内部分提示信息以当地语言呈现___

### __使用方式__
> ___更详细的使用方式请 [前往B站](https://space.bilibili.com/483822869) 观看视频讲解___
- __准备一个文件夹（尽可能不要在系统目录下，会因权限问题写不了文件），将程序文件置于该文件夹下__
- __启动程序文件，配置好参数，点击 `GO` 按钮__
- __程序会提示操作是否完成，提示窗口下方有直达链接，通过该链接可以直接在浏览器中打开 `NVIDIA` 官网，并尝试启动 `DevTools` 工具__
- __若首次使用本程序，需执行以下操作__
   - __在 `DevTools` 工具窗口中找到「源代码」选项卡__
   - __进入后点击页面上的「覆盖」页__
   - __勾选「启用本地替代」，并将缓存文件夹设置为程序所在文件夹__
   - __关闭网页，并重新通过程序窗口中的链接访问 `NVIDIA` 官网__
- ___如程序未能唤起网页，可手动启动浏览器进入 [`www.nvidia.cn/drivers`](https://www.nvidia.cn/drivers/) ，并手动开启浏览器 `DevTools` 工具___
- __在维持 `DevTools` 工具开启的情况下，直接在页面搜索对应的驱动程序，等待搜索结果即可__
- __如后续搜索结果页面中，浏览器 `DevTools` 工具的控制台页面不再产生新的日志，且最后一条显示 `Timeout` ，可尝试刷新页面，此情况为链接至NV服务器超时，并非程序问题__
